# 患者管理产品需求文档

> **模块**：患者管理  
> **角色**：role:product  
> **来源**：Root Issue [gjyhj1234/autoagents3#7](https://github.com/gjyhj1234/autoagents3/issues/7)  
> **状态**：草稿

---

## 1. 业务目标

实现一套完整的患者信息管理功能，支持医疗机构对患者基本信息进行全生命周期管理，包括新增、查询、修改、删除及详情查看，提升医疗数据管理效率与准确性。

---

## 2. 功能范围

| 功能模块 | 说明 |
|---|---|
| 患者信息新增 | 录入患者基本档案 |
| 患者信息查询 | 支持分页列表与多条件筛选 |
| 患者信息修改 | 编辑已有患者档案 |
| 患者信息删除 | 软删除或物理删除患者记录 |
| 患者信息详情 | 查看单个患者的完整档案 |

---

## 3. 功能需求

### 3.1 患者信息新增

- **触发方式**：用户点击「新增患者」按钮，弹出新增表单。
- **必填字段**：
  - 姓名（中文或英文，2–50 个字符）
  - 性别（男 / 女 / 其他）
  - 出生日期（日期选择器，不可晚于当天）
  - 联系电话（手机号或固话，格式校验）
  - 身份证号（18 位，可选，唯一；需校验格式：前 17 位数字 + 第 18 位校验码，并通过 Luhn 加权求和算法验证）
- **选填字段**：
  - 家庭住址
  - 紧急联系人姓名
  - 紧急联系人电话
  - 备注
- **提交规则**：
  - 必填项缺失时前端实时提示，不允许提交。
  - 身份证号重复时后端返回错误，前端提示「该身份证号已存在」。
  - 成功后列表自动刷新，并定位到新增记录。

### 3.2 患者信息查询（分页 + 筛选）

- **列表默认展示字段**：序号、姓名、性别、出生日期、联系电话、创建时间、操作按钮。
- **分页**：
  - 默认每页 20 条，可选 10 / 20 / 50 / 100。
  - 显示总记录数与当前页码。
- **筛选条件**（任意组合）：
  - 姓名（模糊匹配）
  - 性别（单选下拉）
  - 出生日期范围（日期区间选择器）
  - 联系电话（精确或前缀匹配）
  - 创建时间范围
- **排序**：默认按创建时间倒序，支持点击列头排序（创建时间、姓名）。
- **导出**：支持将当前筛选结果导出为 CSV/Excel（不超过 5000 条）。

### 3.3 患者信息修改

- **触发方式**：列表行内「编辑」按钮，或详情页「编辑」按钮。
- **可修改字段**：与新增表单一致（姓名、性别、出生日期、联系电话、身份证号、家庭住址、紧急联系人、备注）。
- **不可修改字段**：患者 ID、创建时间、创建人。
- **提交规则**：
  - 字段校验规则与新增一致。
  - 修改后记录最后修改时间与修改人。
  - 成功后详情/列表自动刷新。

### 3.4 患者信息删除

- **触发方式**：列表行内「删除」按钮，或详情页「删除」按钮。
- **交互**：
  - 弹出二次确认弹窗，显示「确认删除患者 <姓名>？此操作不可恢复。」
  - 用户确认后执行删除。
- **删除策略**：
  - 默认采用**软删除**（标记 `deleted_at` 时间戳），数据不物理移除，保留审计追踪。（物理删除需由超级管理员在后台手动操作）
  - 已被医嘱或就诊记录引用的患者不允许删除，提示「该患者存在关联数据，无法删除」。
- **批量删除**：列表支持多选，点击「批量删除」一次最多删除 50 条。

### 3.5 患者信息详情

- **触发方式**：列表行内「查看」按钮，或点击患者姓名。
- **展示内容**：
  - 所有基本信息字段（含只读的创建时间、创建人、最后修改时间、修改人）。
  - 关联就诊记录摘要（若模块已开发，否则占位显示「暂无数据」）。
- **操作按钮**：详情页提供「编辑」与「删除」快捷按钮。

## 4. 非功能需求

| 类别 | 要求 |
|---|---|
| 性能 | 列表查询响应时间 ≤ 500 ms（数据量 ≤ 100 万条） |
| 安全 | 所有接口需登录鉴权；敏感字段（身份证号、联系电话）在日志与响应体中脱敏 |
| 可用性 | 服务可用率 ≥ 99.9%（月均） |
| 兼容性 | 支持 Chrome 最新版、Edge 最新版、Firefox 最新版；移动端响应式布局 |
| 国际化 | 优先支持简体中文；界面文案支持多语言扩展 |
| 数据完整性 | 所有写操作使用数据库事务，保证原子性 |

---

## 5. 验收标准

1. 可成功新增一条患者记录，所有必填字段均经过前后端双重校验。
2. 列表页可按姓名模糊搜索，结果正确分页展示，切换页码数据不重复。
3. 筛选条件任意组合后，返回结果与数据库实际数据一致。
4. 可成功修改患者信息，修改时间与修改人字段自动更新。
5. 软删除后，前端列表不再展示该记录，数据库中 `deleted_at` 字段已赋值。
6. 存在关联数据的患者删除时，系统正确拦截并给出提示，记录不被删除。
7. 详情页展示内容与数据库记录一致（敏感字段除外，已按规则脱敏显示），身份证号按 `310***********1234` 格式脱敏，联系电话按 `138****8888` 格式脱敏。
8. 导出 CSV/Excel 内容与当前筛选结果一致，文件可正常打开。

## 6. 术语表

| 术语 | 说明 |
|---|---|
| 软删除 | 在记录上标记删除时间戳，不物理移除数据 |
| 脱敏 | 对敏感信息进行部分遮蔽，如 `310***********1234` |
| 分页 | 将列表数据分批返回，每批称为一页 |

# 患者管理模块 — 产品需求文档

## 1. 背景与目标

本模块为医疗信息系统的核心基础功能，实现患者基本信息的统一管理，
支持临床、护理、行政人员快速录入、检索和维护患者档案，提升医院运营效率
与数据准确性。

## 2. 角色与权限

| 角色     | 新增 | 查询 | 修改 | 删除 | 查看详情 |
|----------|------|------|------|------|----------|
| 管理员   | ✅   | ✅   | ✅   | ✅   | ✅       |
| 医护人员 | ✅   | ✅   | ✅   | ❌   | ✅       |
| 访客     | ❌   | ✅   | ❌   | ❌   | ✅       |

## 3. 功能清单

### 3.1 患者信息新增

- **入口**：患者列表页右上角「新增患者」按钮。
- **必填字段**：姓名、性别、出生日期、联系电话。
- **选填字段**：身份证号、地址、过敏史、备注。
- **规则**：
  - 手机号格式校验（11 位数字）。
  - 身份证号唯一性校验；若已存在则提示「患者已存在」。
  - 提交成功后自动跳转到患者详情页。

### 3.2 患者信息查询（分页 + 筛选）

- **默认视图**：按创建时间倒序分页展示，每页 20 条。
- **筛选项**：姓名（模糊）、性别、手机号（精确）。
- **分页**：支持上一页 / 下一页 / 跳转至指定页。
- **响应时间**：≤ 500 ms（10 万条数据规模）。

### 3.3 患者信息修改

- **入口**：列表行「编辑」按钮 或 详情页「编辑」按钮。
- **可编辑字段**：除 ID 外所有字段。
- **规则**：修改手机号时重新做格式校验；修改身份证号时重新做唯一性校验。
- **操作日志**：记录修改人、修改时间、修改前后值（审计要求）。

### 3.4 患者信息删除

- **入口**：列表行「删除」按钮（仅管理员可见）。
- **二次确认**：弹窗提示「确认删除该患者？此操作不可恢复。」。
- **软删除**：标记 `is_deleted=true`，数据保留用于审计；查询接口默认过滤。

### 3.5 患者信息详情

- **入口**：列表行患者姓名链接。
- **展示内容**：所有字段 + 创建时间 + 最后修改时间。

## 4. 非功能需求

- API 接口遵循 RESTful 规范，返回统一 JSON 格式。
- 所有接口支持 HTTPS。
- 前端适配桌面 Chrome / Firefox / Edge 最新版本。

## 5. 验收标准

- [ ] 能够成功新增患者并在列表中展示。
- [ ] 分页查询正常工作，筛选条件生效。
- [ ] 编辑患者信息后，详情页实时更新。
- [ ] 删除操作需二次确认，删除后列表不再显示该患者。
- [ ] 患者详情页显示完整信息。

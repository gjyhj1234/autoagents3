# 患者管理产品需求文档

**版本**：v1.0  
**日期**：2026-04-21  
**角色**：role:product  
**关联根 Issue**：[#15 [ROOT] 患者管理](https://github.com/gjyhj1234/autoagents3/issues/15)

---

## 1. 业务目标

实现患者管理的全部功能，为医疗机构提供完整的患者信息生命周期管理能力，包括患者档案的新增、查询、修改、删除及详情查看，确保数据准确、操作高效、权限可控。

---

## 2. 用户角色

| 角色 | 描述 |
|------|------|
| 医护人员 | 日常录入、查询、修改患者信息的主要操作者 |
| 管理员 | 拥有删除等高权限操作的系统管理人员 |

---

## 3. 功能需求

### 3.1 患者信息新增

**描述**：允许医护人员录入新患者的基本信息，建立患者档案。

**基本信息字段**：

| 字段 | 类型 | 是否必填 | 说明 |
|------|------|----------|------|
| 姓名 | 字符串 | 是 | 患者真实姓名，最长 50 字符 |
| 性别 | 枚举 | 是 | 男 / 女 / 未知 |
| 出生日期 | 日期 | 是 | 格式 YYYY-MM-DD |
| 身份证号 | 字符串 | 否 | 18 位，唯一校验 |
| 手机号码 | 字符串 | 否 | 11 位，格式校验 |
| 联系地址 | 字符串 | 否 | 最长 200 字符 |
| 紧急联系人 | 字符串 | 否 | 姓名 + 手机号 |
| 备注 | 文本 | 否 | 最长 500 字符 |

**验收标准**：

- [ ] 必填字段为空时，表单提示错误，无法提交。
- [ ] 身份证号重复时，提示"患者已存在"并阻止新增。
- [ ] 提交成功后，系统自动生成唯一患者编号（Patient ID），并展示在详情页。
- [ ] 新增成功后跳转至患者详情页。

---

### 3.2 患者信息查询（分页、筛选）

**描述**：支持以分页列表形式展示所有患者，并提供多条件筛选功能。

**筛选条件**：

| 条件 | 类型 | 说明 |
|------|------|------|
| 患者姓名 | 模糊搜索 | 支持关键字部分匹配 |
| 患者编号 | 精确搜索 | 按系统生成的 Patient ID 查询 |
| 性别 | 下拉选择 | 男 / 女 / 全部 |
| 出生日期范围 | 日期区间 | 开始日期 ~ 结束日期 |
| 手机号码 | 精确搜索 | |

**分页规格**：

- 默认每页展示 20 条，支持切换为 10 / 50 / 100 条。
- 展示总条数与当前页码。

**列表字段**：患者编号、姓名、性别、出生日期、手机号码、创建时间、操作（查看 / 编辑 / 删除）。

**验收标准**：

- [ ] 不填任何筛选条件时，返回全部患者列表（分页）。
- [ ] 关键字搜索响应时间 ≤ 500 ms（数据量 ≤ 10 万条）。
- [ ] 翻页后保持筛选条件不变。
- [ ] 列表支持按创建时间降序默认排序。

---

### 3.3 患者信息修改

**描述**：允许医护人员对已有患者档案进行编辑更新。

**可修改字段**：除患者编号（系统生成，不可改）外，3.1 中所有字段均可修改。

**验收标准**：

- [ ] 进入编辑页时，所有字段预填当前值。
- [ ] 身份证号修改后，仍需通过唯一性校验（排除本条记录）。
- [ ] 保存成功后展示成功提示，并返回详情页。
- [ ] 每次修改记录操作日志（操作人、操作时间、修改前后值）。

---

### 3.4 患者信息删除

**描述**：管理员可删除患者档案，支持单条删除。

**验收标准**：

- [ ] 删除前弹出二次确认弹窗，提示"删除后不可恢复"。
- [ ] 确认后执行软删除（逻辑删除，数据库保留记录，`deleted_at` 标记时间戳）。
- [ ] 删除成功后，该患者不再出现在查询列表中。
- [ ] 普通医护人员无删除权限，按钮置灰或不显示。

---

### 3.5 患者信息详情

**描述**：展示单个患者的完整档案信息及操作历史。

**详情页展示内容**：

- 患者基本信息（3.1 全部字段）。
- 患者编号、创建时间、最后修改时间。
- 操作日志列表（时间 / 操作人 / 操作类型 / 变更摘要）。

**验收标准**：

- [ ] 通过患者编号或列表"查看"按钮均可进入详情页。
- [ ] 页面加载时间 ≤ 1 s。
- [ ] 详情页提供"编辑"快捷入口。

---

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 列表查询 P99 响应时间 ≤ 1 s（10 万条数据规模） |
| 安全 | 所有接口需登录鉴权；删除接口需管理员角色 |
| 数据完整性 | 软删除，历史数据可追溯 |
| 可用性 | 服务可用率 ≥ 99.9% |
| 兼容性 | 支持 Chrome 最新版、Edge 最新版 |

---

## 5. 交互原则

1. 表单校验采用实时（blur）+ 提交双重校验。
2. 危险操作（删除）必须有二次确认。
3. 操作成功 / 失败均有明确的 Toast 提示。
4. 列表页筛选条件支持一键重置。

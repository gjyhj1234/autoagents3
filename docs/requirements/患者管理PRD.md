> **版本**：v1.0  
> **状态**：Draft  
> **关联 Issue**：[ROOT] 患者管理 #15

---

## 1. 背景与业务目标

本模块旨在实现**患者管理**的全部功能，为医疗机构提供完整的患者信息生命周期管理能力。系统需支持患者信息的录入、查询、修改、删除及详情查看，以提升医务人员的工作效率，确保患者数据准确、安全、可追溯。

---

## 2. 功能范围

| 序号 | 功能模块         | 优先级 | 说明                                           |
|------|------------------|--------|------------------------------------------------|
| F-01 | 患者信息新增     | P0     | 录入新患者的基本信息                           |
| F-02 | 患者信息查询     | P0     | 支持分页浏览与多条件筛选                       |
| F-03 | 患者信息修改     | P0     | 编辑已有患者的信息字段                         |
| F-04 | 患者信息删除     | P1     | 逻辑删除（软删除），保留历史记录               |
| F-05 | 患者信息详情     | P0     | 查看单个患者的完整信息                         |

## 4. 功能需求详情

### 4.1 F-01 患者信息新增

**描述**：允许医护人员录入新患者的基本信息，生成唯一患者 ID。

**输入字段**：

| 字段名       | 类型     | 必填 | 说明                           |
|--------------|----------|------|--------------------------------|
| 姓名         | 字符串   | 是   | 患者真实姓名，最长 50 字符      |
| 性别         | 枚举     | 是   | 男 / 女 / 未知                 |
| 出生日期     | 日期     | 是   | 格式 YYYY-MM-DD                |
| 身份证号     | 字符串   | 否   | 18 位，需通过格式校验           |
| 手机号       | 字符串   | 是   | 11 位国内手机号                |
| 家庭住址     | 字符串   | 否   | 最长 200 字符                  |
| 过敏史       | 文本     | 否   | 自由文本，最长 500 字符         |
| 紧急联系人   | 字符串   | 否   | 姓名 + 手机号                  |
| 备注         | 文本     | 否   | 最长 1000 字符                 |

**业务规则**：
- 手机号在系统中唯一，重复时提示「该手机号已注册患者」。
- 身份证号若填写则校验格式与校验码。
- 新增成功后生成系统唯一患者 ID（`PAT-YYYYMMDD-XXXX`）。

**验收标准**：
- [ ] 必填字段为空时，表单提示具体错误信息，无法提交。
- [ ] 重复手机号提交时，显示错误提示并阻止新增。
- [ ] 成功新增后页面跳转至患者详情页，并显示新生成的患者 ID。

---

### 4.2 F-02 患者信息查询（分页、筛选）

**描述**：提供患者列表页，支持关键词搜索与多维度筛选，结果分页展示。

**筛选条件**：

| 条件       | 类型   | 说明                           |
|------------|--------|--------------------------------|
| 姓名       | 模糊   | 支持中文模糊搜索               |
| 手机号     | 精确   | 11 位精确匹配                  |
| 身份证号   | 精确   | 精确匹配                       |
| 性别       | 枚举   | 男 / 女 / 全部                 |
| 创建日期   | 区间   | 开始日期 ~ 结束日期             |

**分页规则**：
- 默认每页 20 条，可选 10 / 20 / 50。
- 总数超过 10000 时显示「约 1 万条」。

**验收标准**：
- [ ] 无筛选条件时，列表按创建时间降序展示所有患者。
- [ ] 多条件同时筛选时，结果为所有条件的交集（AND 逻辑）。
- [ ] 分页切换后筛选条件保持不变。
- [ ] 列表响应时间 ≤ 500ms（数据量 ≤ 10 万条）。

---

### 4.3 F-03 患者信息修改

**描述**：允许医护人员修改已有患者的信息字段，并记录操作日志。

**可修改字段**：4.1 节中除「患者 ID」与「创建时间」之外的所有字段。

**业务规则**：
- 修改手机号时，新手机号不得与其他患者重复。
- 每次修改记录操作人、操作时间和修改前后的变化（审计日志）。

**验收标准**：
- [ ] 修改成功后，详情页立即反映最新数据。
- [ ] 修改手机号为已存在号码时，提示冲突并阻止保存。
- [ ] 操作日志可在详情页「修改记录」标签中查看。

---

### 4.4 F-04 患者信息删除

**描述**：逻辑删除（软删除）患者记录，保留数据用于审计，不可物理删除。

**业务规则**：
- 仅管理员可执行删除操作。
- 删除前弹出二次确认弹窗。
- 被删除患者不出现在正常查询列表中，但可在「回收站」中查看。
- 已删除患者可由管理员恢复。

**验收标准**：
- [ ] 普通医护人员无删除按钮，管理员才可见。
- [ ] 删除后患者从主列表消失，在回收站可找到。
- [ ] 回收站支持恢复与彻底删除（彻底删除需二次确认，且仅限超级管理员）。

---

### 4.5 F-05 患者信息详情

**描述**：展示单个患者的完整信息，包括基本信息、就诊记录和修改日志。

**展示内容**：

| 分区         | 内容                                             |
|--------------|--------------------------------------------------|
| 基本信息     | 患者 ID、姓名、性别、出生日期、年龄（自动计算）、身份证号、手机号、家庭住址、过敏史、紧急联系人、备注 |
| 就诊记录     | 预留（由后续 role:architect / role:backend 补充）|
| 修改记录     | 操作人、操作时间、修改字段及前后值               |

**验收标准**：
- [ ] 详情页加载时间 ≤ 300ms。
- [ ] 年龄根据出生日期实时计算，格式为「XX 岁」。
- [ ] 修改记录按时间降序排列，最多展示最近 50 条。

---

## 5. 非功能需求

| 类型       | 要求                                                     |
|------------|----------------------------------------------------------|
| 性能       | 列表页查询 ≤ 500ms，详情页加载 ≤ 300ms                   |
| 安全性     | 接口需鉴权（JWT），患者数据传输使用 HTTPS 加密           |
| 可用性     | 服务可用性 ≥ 99.9%（月均）                               |
| 数据合规   | 患者敏感字段（身份证号、手机号）存储时脱敏或加密         |
| 审计       | 所有增删改操作记录操作人与时间戳，日志保留 ≥ 3 年       |
| 国际化     | 当前仅需支持简体中文                                     |

## 6. 数据模型（概览）

> 详细 Schema 由 `role:architect` 阶段产出，此处仅列出核心字段供参考。

```
Patient {
  id            String   // 系统唯一 ID，格式 PAT-YYYYMMDD-XXXX
  name          String
  gender        Enum     // MALE | FEMALE | UNKNOWN
  birthDate     Date
  idCard        String?  // 脱敏存储
  phone         String   // 唯一索引
  address       String?
  allergyInfo   String?
  emergencyContact String?
  remark        String?
  isDeleted     Boolean  // 软删除标志
  createdAt     DateTime
  updatedAt     DateTime
  createdBy     String   // 操作人 ID
}
```
